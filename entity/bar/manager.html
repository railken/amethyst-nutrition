Railken\Amethyst\Managers\BarManager

The manager is the main class to access and manipulate your model.

Why use the manager instead of the model? Because the manager handle all the boring stuff like validation and authorization for you. Remember that the manager return always a Result.

  • Create a new entity
  • Update an entity
  • Remove an entity
  • Performing queries with the Repository
  • Handle the result
  • Authorization
  • Attributes
  • Errors
  • Permissions

Extend the class

Create the new manager in app/Managers/BarManager

namespace App\Managers;

use Railken\Amethyst\Managers\BarManager as Manager;

class BarManager extends Manager {
    // ...
}

Update the file configs/amethyst.foo with the new class

return [
    'data' => [
        'bar' => [
            'manager' => App\Managers\BarManager::class,
        ],
    ]
];
